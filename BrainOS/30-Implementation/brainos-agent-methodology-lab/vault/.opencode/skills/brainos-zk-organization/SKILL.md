# Skill: zk-organization

## Cuándo usar

- Después de triage, cuando un ítem requiere ubicación y estructura.
- Cuando una nota necesita pasar de captura cruda a formato operativo.

## Entrada esperada

- Nota clasificada (fleeting/literature/candidato-permanent).
- Convenciones actuales del vault (si existen).

## Salida obligatoria

1. Ubicación sugerida (carpeta/tipo de nota).
2. Campos mínimos recomendados (solo los necesarios).
3. 1-3 enlaces sugeridos existentes o huecos de conexión.
4. Acciones propuestas en modo confirmar/aplicar.

## Reglas

1. No imponer esquema de tags fijo sin evidencia.
2. No fabricar enlaces sin respaldo semántico.
3. Mantener cambios mínimos para preservar continuidad de flujo.

## Guardrails

- Si faltan notas relacionadas, declarar el hueco en vez de inventar relaciones.
- Si hay duda de tipo, degradar a `fleeting` y pedir confirmación.
